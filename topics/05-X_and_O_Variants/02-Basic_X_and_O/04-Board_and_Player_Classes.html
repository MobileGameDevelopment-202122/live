Board and Players

The Board class performs a similar role to the level class in BunnyHop.

In order to support (later) any combination of human and computer players we have a number of classes representing players:

  • BasePlayer - the base class for all players.
  • HumanPlayer - first implementation of a human player.
  • FirstSpacePlayer - a computer player which uses the very very poor strategy of using the first empty cell it finds.

A move can be represented by a pair of integers (the row and column coordinates), however, when passing back a move as a single return value we need to either wrap the pair of __int__s into an object or encode the pair of integers into a single integer. I prefer the encoding strategy, and use the following encoding

encoding (row,col) -> pos

1
pos = row*3+col;

encoding pos -> (row,col)

1
2
row = pos / 3;
col = pos % 3;

    public boolean isDraw()

Method: hasWon

1
    public boolean hasWon(int symbol, int row, int col)
  • true if the player using symbol symbol has won after placing at position (row, col)
  • Take care to ensure you understand the test that hasWon preforms. Rather than testing if a board contains a winning line (column/row/diagonal) for either player, it simply checks if that last move results in a win. Hence we can avoid checking the entire board (this is a big deal with we move to games with larger boards i.e. connect 4).
